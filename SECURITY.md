# Security Policy

## Supported Versions

The following versions of DevTrack are currently receiving security updates:

| Version | Supported          |
| ------- | ------------------ |
| Latest (`main` branch) | ✅ Supported |
| Older versions          | ❌ Not supported |

> We recommend always using the latest version of DevTrack to ensure you have all security patches.

---

## Reporting a Vulnerability

We take security vulnerabilities seriously and appreciate the efforts of security researchers and the community in helping us keep DevTrack safe.

**Please do not report security vulnerabilities through public GitHub Issues.**

### GitHub Private Vulnerability Reporting

Use GitHub's built-in private reporting feature:

1. Go to the **Security** tab of this repository.
2. Click **"Report a vulnerability"** under the Private Reporting section.
3. Fill in the details of the vulnerability and submit.

This keeps the report confidential and allows us to collaborate on a fix before any public disclosure.

### What to Include in Your Report

When reporting a vulnerability, please include:

- A clear description of the issue
- Steps to reproduce the vulnerability
- Potential impact
- Screenshots, logs, or proof-of-concept code (if applicable)
- Suggested remediation (optional)

---

## Expected Response Time

| Stage | Timeframe |
| ----- | --------- |
| Initial acknowledgement | Within **48 hours** |
| Vulnerability assessment | Within **7 days** |

We follow responsible disclosure principles. If you have specific timing requirements, please mention them in your report.

---

## What Happens After a Report Is Received

1. The maintainers will acknowledge receipt of the report.
2. The vulnerability will be reviewed and validated.
3. Additional information may be requested if needed.
4. A fix will be developed and tested.
5. Security updates will be released for supported versions.
6. The reporter may be informed once the issue has been resolved.

---

## Scope

The following are **in scope** for vulnerability reports:

- The DevTrack web application (frontend and backend)
- Authentication and authorization flaws
- Data exposure or injection vulnerabilities
- Dependency vulnerabilities with a direct exploitable impact
